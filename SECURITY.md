# Security Policy

## Reporting a Vulnerability

Please report vulnerabilities privately through your repository's security advisory process.

Include:

- Affected version(s)
- Reproduction steps / proof of concept
- Impact assessment
- Suggested mitigation (if known)

## Supported Versions

Only the latest published package version is guaranteed to receive security fixes.

## Response Targets

- Initial triage: within 3 business days
- Status update: within 7 business days
- Patch timeline: based on severity and exploitability
